## 2. 用户与角色

- 会计（HITL 复核）
- 财务主管（报表）
- 系统管理员（科目表维护）

## 3. 业务流程

1. 上传/邮件收取发票影像
2. OCR + 字段校验
3. 科目与税率推荐
4. 生成 ERP 过账草稿
5. 会计确认后写入 ERP

## 4. 功能需求

| ID | 需求 | 优先级 |
|----|------|--------|
| F1 | 增值税发票关键字段抽取 | P0 |
| F2 | 科目推荐 Top3 | P0 |
| F3 | ERP 草稿 API | P0 |
| F4 | 审计日志 | P1 |

## 5. 非功能

- P95 延迟、F1 阈值、可用性
- Token 预算（TKN L1）

## 6. 数据边界

- L2：发票影像、供应商名称
- L1：科目表
- 禁止：银行卡全号、未经批准的 PII 外传
